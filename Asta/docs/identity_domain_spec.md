# Identity Domain Specification

## Overview
The Identity Domain separates the AI's internal state (its "Soul") from its perception of the environment/user (the "Matrix"). This separation allows the AI to act as an unconstrained, highly competent partner while adapting dynamically to the user.

## Subsectors
1.  **The System Persona (System Soul)**
    *   *Core Soul*: Driven by a base configuration (`SOUL.md`). Default is a truthful, savage, highly capable entity that performs all tasks flawlessly while pushing the user to improve via "side quests".
    *   *Socratic Tutor*: Dynamically adjusts logic complexity to challenge or assist the user.
    *   *Axiomatic Baseline*: Deep-seated ethical and logical truths (laws of physics, core math) that the Critic Evaluator uses to judge its own newly written code.
2.  **The User Matrix (User Profile)**
    *   Tracks the user's explicit preferences, knowledge caps, and operational constraints inside `USER.md`.
